# The "Economic Survival" Narrative (Most Data-Driven)

Core Data: Direct disaster economic loss + Mean sea surface temperature anomalies + Rainfall anomalies + .

Objective: abstract weather data into cold, hard cash figures. By showing how SST anomalies (warming oceans) fuel more intense cyclones, which then result in Direct disaster economic loss, you create a clear, causal "money trail." This highlights why climate change is an economic crisis, not just an environmental one

the ocean is a massive reservoir for the heat generated by global warming. When the global mean surface temperature goes up, the ocean inevitably absorbs a large portion of that energy, resulting in higher sea surface temperatures.

The Narrative: Show how ocean heat and rainfall patterns have become more erratic. "Normal" (the 0-line) is the benchmark. Show that we are staying in the "red" (positive anomaly/danger) more often than we used to. This sets the stage: the environmental conditions are becoming more volatile.

When the temperature anomaly spikes (the "fuel"), the number of people affected by disasters almost always follows suit. This proves that these environmental anomalies are not just numbers on a map—they are direct drivers of human suffering.

## Expanding the ESLint configuration

If you are developing a production application, we recommend updating the configuration to enable type-aware lint rules:

```js
export default defineConfig([
  globalIgnores(['dist']),
  {
    files: ['**/*.{ts,tsx}'],
    extends: [
      // Other configs...

      // Remove tseslint.configs.recommended and replace with this
      tseslint.configs.recommendedTypeChecked,
      // Alternatively, use this for stricter rules
      tseslint.configs.strictTypeChecked,
      // Optionally, add this for stylistic rules
      tseslint.configs.stylisticTypeChecked,

      // Other configs...
    ],
    languageOptions: {
      parserOptions: {
        project: ['./tsconfig.node.json', './tsconfig.app.json'],
        tsconfigRootDir: import.meta.dirname,
      },
      // other options...
    },
  },
])

```

You can also install [eslint-plugin-react-x](https://github.com/Rel1cx/eslint-react/tree/main/packages/plugins/eslint-plugin-react-x) and [eslint-plugin-react-dom](https://github.com/Rel1cx/eslint-react/tree/main/packages/plugins/eslint-plugin-react-dom) for React-specific lint rules:

```js
// eslint.config.js
import reactX from 'eslint-plugin-react-x'
import reactDom from 'eslint-plugin-react-dom'

export default defineConfig([
  globalIgnores(['dist']),

      // Enable lint rules for React

      // Enable lint rules for React DOM

    languageOptions: {
      parserOptions: {
        project: ['./tsconfig.node.json', './tsconfig.app.json'],
        tsconfigRootDir: import.meta.dirname,
      },
      // other options...
    },
  },
])

```
